1. Visão Geral

Este projeto é um Dashboard de Performance Operacional focado na busca do Equilíbrio Ideal entre Tempo Médio de Atendimento (TMO) e Qualidade (NPS). O conceito central é identificar a faixa de tempo onde a operação é mais eficiente sem prejudicar a satisfação do cliente.

2. Estrutura de Dados

Granularidade: Caso Individual (Case ID)

Diferente de versões anteriores, a base processada é analítica (linha por atendimento). Toda agregação por faixas de TMO é feita dinamicamente no frontend.

interface DashboardData {
  CASE_ID?: string;
  COLA: string;
  USER_FAIXA_ORDEM: number;           // Ordem numérica da faixa
  USER_FAIXA_HISTOGRAMA: string;      // Label da faixa (ex: "1800 - 1900")
  TMO_SEC: number;                    // Duração do atendimento
  SILENCE_DURATION_SEC: number;       // Silêncio total
  QTD_PESQUISAS_PARA_PIVOT: number;   // 1 ou 0 (Evita inflação de NPS)
  NPS_PONDERADO_PARA_PIVOT: number;   // Score real (Promotores - Detratores)
  META: number;                       // Meta de TMO da fila
  META_NPS: number;                   // Meta de NPS da fila
  PERIODO: string;                    // "CONSOLIDADO" ou "D-1"
  USER_LDAP: string;
  ASSIGN_CI_CURRENT_CHANNEL?: string; // Canal (Chat, Voz, etc)
}

3. Lógica de Negócio: Score de Equilíbrio

O Score de Equilíbrio é a métrica mestra calculada em src/lib/analysis.ts. Ele utiliza uma ponderação tripartite (0.33 cada):

  • TMO Invertido: Normaliza o TMO para que valores menores tenham score maior (Eficiência).
  • NPS Normalizado: Normaliza o NPS médio da faixa (Qualidade).
  • Volume Normalizado: Faixas com maior representatividade estatística ganham peso (Confiança).
Nota Crítica: Somente faixas com pelo menos uma pesquisa de NPS (surveys > 0) são candidatas ao TMO Ideal.
